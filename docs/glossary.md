# Glossary

## Terms and Acronyms

### A

**ARIA (Accessible Rich Internet Applications)**
- Set of attributes for making web content accessible. Used throughout components.

**Axios**
- Promise-based HTTP client for making API requests.

**API (Application Programming Interface)**
- Backend REST API endpoints consumed by the frontend.

### B

**Board**
- Main container for organizing tasks. Displayed in BoardList and Board components.

**Build**
- Process of compiling and bundling source code into production-ready files.

### C

**Component**
- Reusable React component. Building blocks of the UI.

**CORS (Cross-Origin Resource Sharing)**
- Mechanism allowing frontend to make requests to backend API.

**CSS-in-JS**
- Not used. Tailwind CSS provides utility classes instead.

### D

**Drag and Drop**
- HTML5 API for moving elements. Used for task movement between statuses.

**DTO (Data Transfer Object)**
- Data structures received from backend API.

### E

**ES Modules**
- Modern JavaScript module system. Used throughout the project.

### H

**HMR (Hot Module Replacement)**
- Vite feature updating modules without full page reload during development.

**Hook**
- React function for using state and lifecycle features (e.g., `useState`, `useEffect`).

### J

**JSX (JavaScript XML)**
- Syntax extension for writing React components. Looks like HTML in JavaScript.

**JWT (JSON Web Token)**
- Token format used for authentication. Stored in localStorage.

### L

**LocalStorage**
- Browser API for persistent storage. Used for authentication tokens.

### M

**Mermaid**
- Diagramming language used in documentation for flowcharts and sequence diagrams.

### N

**Nginx**
- Web server used in production container to serve static files and proxy API requests.

### P

**PostCSS**
- Tool for transforming CSS. Used by Tailwind CSS.

**Props**
- Properties passed to React components. Data flows from parent to child.

### R

**React**
- JavaScript library for building user interfaces.

**React Router**
- Library for client-side routing in React applications.

**Responsive Design**
- Design approach adapting layout to different screen sizes.

### S

**SPA (Single Page Application)**
- Web application loading a single HTML page and updating content dynamically.

**State**
- Component data that can change. Managed with `useState` hook.

**Status**
- Column in a board representing task state. Tasks belong to a status.

### T

**Tailwind CSS**
- Utility-first CSS framework. Provides utility classes for styling.

**Task**
- Individual work item displayed as a card. Belongs to a status and board.

### V

**Vite**
- Build tool and development server. Provides fast HMR and optimized builds.

**Vitest**
- Testing framework compatible with Vite. Recommended for testing.

### W

**WCAG (Web Content Accessibility Guidelines)**
- Standards for web accessibility. Project targets WCAG 2.1 Level AA.

## Domain-Specific Terms

**OpenFlow**
- Project name for the Trello-like application.

**Status Column**
- Vertical column in board view containing tasks of a specific status.

**Task Card**
- Visual representation of a task. Can be dragged between status columns.

**User Menu**
- Dropdown menu in header showing user information and logout option.

**Protected Route**
- Route requiring authentication. Redirects to login if not authenticated.
